# Hands-on 5 – Autoevaluación

En esta quinta entrega hemos aprendido a identificar, de entre nuestro conjunto de datos, qué clases son buenos candidatos para enlazarlos con otros datasets, a usar los servicios de reconciliación de OpenRefine para mapear valores, actualizar los mapeos YARRRML y RML para generar triples que conecten nuestros recursos con los externos, a volver a materializar el grafo RDF y a escribir consultas SPARQL que aprovechen dichos enlaces mencionados anteriormente.

## Archivos subidos

- Un fichero JSON con las operaciones realizadas a los datos para unirlos (`hospitales-with-links.json`).
- Un fichero CSV con la versión actualizada de los datasets (`hospitales-with-links.csv`).
- Un fichero RML con las versiones actualizadas de los mapeos (`hospitales-with-links.rml.ttl`).
- Un fichero RDF con los datos transformados en RDF (`hospitales-with-links.ttl`).
- Un fichero SPARQL con las queries que verifican nuestros links (`queries-wth-links.sparql`).

## Conclusión

En definitiva, gracias a esta práctica hemos comprendido mejor el paso de **linking**, es decir, cómo pasar de tener solo nuestros datos en RDF a tener un dataset conectado con otros conjuntos de datos abiertos.
